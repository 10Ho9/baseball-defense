# 03. 번트/스퀴즈

이 문서는 `번트/스퀴즈` 카테고리의 원본 문서입니다.

- 수정이 필요하면 이 문서를 먼저 고칩니다.
- 좌표 구현 원문은 [03-bunts-and-squeezes-layout.md](./03-bunts-and-squeezes-layout.md)를 함께 봅니다.
- 이 카테고리의 시나리오는 총 `6개`입니다.
- 핵심 목표는 `코너 차지`, `P 커버`, `middle rotation`, `Home/1B 전환 판단`을 팀 공통 언어로 고정하는 것입니다.

## 시나리오

### 13. `runner-on-first-bunt-third-base-side`

- 상황: 1루 주자, 3루 쪽 번트
- 팀 콜: 3B가 charge, P가 1루 커버, SS가 3루를 메운다.
- Primary Out: `1B`
- Fallback: `2B 선행 아웃이 명확할 때만 전환`
- 핵심 움직임:
  - `3B`는 공격적으로 처리하되 1루가 기본이다.
  - `P`는 1루를 확정적으로 가져간다.
  - `SS`는 비는 3루를 바로 메운다.
- 백업/정렬:
  - `LF`는 3루 라인 뒤를 가장 먼저 잡아 charging 3B 뒤 공간과 오버스로우를 받는다.
  - `CF`는 2루 뒤로 짧게 들어와 선행 아웃 전환과 중간 리바운드를 정리한다.
  - `RF`는 1루선 뒤와 우측 인필드 뒤를 정리해 안전한 1루 플레이의 마지막 백업이 된다.
- 흔한 실수:
  - 3B가 2루 욕심을 너무 오래 보는 것

### 14. `runner-on-first-bunt-first-base-side`

- 상황: 1루 주자, 1루 쪽 번트
- 팀 콜: 1B가 charge, P가 1루 커버, 2B가 2루 포스를 맡고 SS가 middle backside를 정리한다.
- Primary Out: `1B`
- Fallback: `2B 선행 아웃`
- 핵심 움직임:
  - `1B`는 타구를 공격하면서도 1루 커버 확인을 먼저 한다.
  - `P`는 1루 커버 각도를 크게 만든다.
  - `2B`는 2루 포스 플레이를 준비한다.
  - `SS`는 middle backside와 중간 리바운드를 정리한다.
- 백업/정렬:
  - `RF`는 1루선 뒤를 가장 먼저 잡아 1루 송구 이탈의 첫 백업이 된다.
  - `CF`는 2루 뒤로 안쪽 이동해 선행 아웃 전환과 가운데 리바운드를 정리한다.
  - `LF`는 좌측 중간과 3루 쪽을 닫아 비는 좌측 공간을 관리한다.
- 흔한 실수:
  - 누가 1루를 가져가는지 모호해지는 것

### 15. `runners-on-first-second-sac-bunt-third-base-side`

- 상황: 1,2루 주자, 3루 쪽 희생번트
- 팀 콜: 3B가 charge해 앞선 주자 3루를 먼저 보고, 닫히면 안정적으로 1루를 잡는다.
- Primary Out: `3B`
- Fallback: `1B`
- 핵심 움직임:
  - `3B`는 리드주자 속도와 타구 질을 동시에 읽는다.
  - `SS`는 3루 커버를 준비한다.
  - `P`는 1루 커버를 놓치지 않는다.
- 백업/정렬:
  - `LF`는 3루 라인 뒤를 깊지 않게 잡아 선행 아웃 송구나 귀루 송구 이탈을 받는다.
  - `CF`는 2루 뒤로 짧게 들어와 3루가 닫힐 때 후속 플레이와 리바운드를 정리한다.
  - `RF`는 1루선 뒤를 준비해 fallback 1루 플레이의 마지막 백업이 된다.
- 흔한 실수:
  - 선행 아웃이 닫혔는데도 늦게 3루로 던지는 것

### 16. `runners-on-first-second-sac-bunt-first-base-side`

- 상황: 1,2루 주자, 1루 쪽 희생번트
- 팀 콜: 1B가 처리하되 3루 선행을 먼저 보고, 늦으면 빠르게 1루 안전 아웃으로 전환한다.
- Primary Out: `3B`
- Fallback: `1B`
- 핵심 움직임:
  - `1B`는 볼을 잡기 전에 선행주자 속도를 체크한다.
  - `P`는 1루 커버를 가져간다.
  - `3B`는 3루 stop-base를 유지해 선행 아웃 창을 만든다.
  - `SS`와 `2B`는 2루 backside와 다음 전환을 분담한다.
- 백업/정렬:
  - `RF`는 1루선 뒤를 먼저 잡아 1루 fallback 이탈구를 정리한다.
  - `CF`는 2루 뒤로 짧게 이동해 2루/3루 전환과 가운데 리바운드를 맡는다.
  - `LF`는 좌측 중간과 3루 뒤 공간을 닫아 선행주자 추가 진루를 막는다.
- 흔한 실수:
  - 선행 아웃 욕심 때문에 아무 아웃도 못 만드는 것

### 17. `runner-on-third-safety-squeeze-defense`

- 상황: 3루 주자, 세이프티 스퀴즈 시도
- 팀 콜: 타구 확인 후 Home을 우선 보되, 무리하지 않으면 1루로 안전 아웃을 만든다.
- Primary Out: `Home if open`
- Fallback: `1B`
- 핵심 움직임:
  - `C`, `P`, `Corners`가 짧은 타구를 즉시 처리한다.
  - `C`는 런너 스타트가 늦는지 빠른지 확인한다.
  - `Middle`은 다음 베이스를 정리한다.
- 백업/정렬:
  - `LF`는 좌측 홈 라인을 짧게 잡아 Home 쪽 이탈구를 첫 번째로 받는다.
  - `RF`는 우측 홈 라인과 1루선 사이를 정리해 fallback 1루 플레이까지 대비한다.
  - `CF`는 얕은 정중앙으로 들어와 플레이트 뒤 큰 리바운드와 다음 베이스 진루를 정리한다.
- 흔한 실수:
  - 런너를 보고 주저하다가 Home과 1루 둘 다 놓치는 것

### 18. `runner-on-third-suicide-squeeze-wheel-play`

- 상황: 3루 주자, 스퀴즈/휠 플레이
- 팀 콜: 내야가 wheel rotation을 돌며 Home 우선 구조를 만든다.
- Primary Out: `Home`
- Fallback: `1B`
- 핵심 움직임:
  - `Corners`는 번트 공격과 홈 라인 차단을 준비한다.
  - `Middle`은 회전해서 비는 베이스를 메운다.
  - `C`는 번트 각도에 따라 직접 처리 또는 수비수에게 타깃을 준다.
- 백업/정렬:
  - `LF`는 3루 라인과 좌측 홈 라인 사이를 정리해 left-side miss를 모두 받는다.
  - `RF`는 1루선과 우측 홈 라인을 정리해 fallback 1루와 Home 이탈구를 함께 대비한다.
  - `CF`는 얕은 정중앙으로 들어와 플레이트 뒤 큰 리바운드와 후속 런다운 시작점을 맡는다.
- 흔한 실수:
  - 휠 회전이 반 박자 늦어 Home이 비는 것
