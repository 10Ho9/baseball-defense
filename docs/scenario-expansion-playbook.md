# 수비 시나리오 확장 플레이북

> 참고: 시나리오 커리큘럼의 정본은 [defense-scenario-curriculum.md](/Users/ho9/develop/baseball/docs/defense-scenario-curriculum.md)입니다. 이 문서는 `36개 정본 밖의 추가 확장`을 검토할 때만 쓰는 운영 메모입니다.

## 문서 목적

이 문서는 아래 두 가지에만 사용합니다.

- 36개 정본 커리큘럼을 모두 구현한 뒤 다음 확장 후보를 고를 때
- 새 후보가 정본 원칙을 지키는지 빠르게 검토할 때

이 문서는 기존 36개 시나리오의 전술 정본을 대체하지 않습니다.

## 이 문서가 다루지 않는 것

- 현재 36개 시나리오의 상세 팀 콜
- 현재 구현된 11개 시나리오의 행동 원문
- 데이터 모델의 최종 구조

위 내용은 각각 아래 문서를 봅니다.

- 전술 정본: `docs/defense-scenario-curriculum.md`, `docs/defense-curriculum/*.md`
- 좌표 정본: `docs/field-layout-reference.md`, `docs/defense-curriculum/*-layout.md`
- 데이터 정본: `docs/scenario-data-model.md`

## 확장 원칙

- 새 시나리오는 `장면 하나`가 아니라 `팀 콜 하나`를 추가한다는 기준으로 검토합니다.
- 한 시나리오에는 하나의 `Primary Out`과 하나의 `Fallback`이 있어야 합니다.
- 같은 게임 상태라도 팀 콜이 다르면 별도 시나리오로 분리합니다.
- 주 처리자, 빈 베이스 커버, 백업, 첫 스텝이 모두 문장으로 고정되지 않으면 아직 후보 단계로 남깁니다.
- 좌우 대칭 장면은 커버와 백업이 달라지므로 필요하면 별도 시나리오로 둡니다.

## 36개 구현 후 1차 확장 후보

- 투수 정면 컴백커 전용 시나리오
- 홈 앞 약한 타구 전용 시나리오
- 2루 주자 도루 시 3루 커버 전용 시나리오
- 1루-2루 런다운 전용 시나리오
- 좌익선상/우익선상 flare 우선권
- RF 얕은 플라이 태그업 Home 송구 전용 시나리오

## 후보 검토 체크리스트

- 기존 36개 안에 사실상 같은 팀 콜이 이미 있는가
- Primary Out과 Fallback이 하나씩 선명한가
- 공을 처리하지 않는 선수의 역할이 더 명확해지는가
- 초보자가 실제로 자주 헷갈리는 장면인가
- 좌표와 애니메이션으로 재현 가능한가
- 퀴즈 한두 문장으로 핵심 판단을 검증할 수 있는가

## 운영 메모

정본 36개를 구현하기 전에는 이 문서로 방향을 다시 흔들지 않습니다.
정본 밖의 후보를 채택하기로 결정하면 먼저 `docs/defense-scenario-curriculum.md`에 편입 여부를 판단한 뒤, 카테고리 원문과 데이터 문서를 순서대로 갱신합니다.
