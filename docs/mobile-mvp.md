# 모바일 MVP 구조

## 화면 원칙

- 한 화면에 너무 많은 제어 요소를 넣지 않습니다.
- 필드 애니메이션을 가장 크게 보여줍니다.
- 긴 세로 스크롤보다 `Ablauf`와 `Position` 모드 전환을 우선합니다.
- 설명을 보기 위해 필드가 화면 아래로 밀려나지 않게 합니다.
- 36개 시나리오를 평면 리스트로 한 번에 보여주지 않고 `카테고리 -> 시나리오` 순서로 진입시킵니다.
- 최종 사용자 문구는 독일어를 기본으로 하되, `Double Play`, `Pop-Up`, `Play at the Plate` 같은 야구 용어는 영어 표기를 유지할 수 있습니다.
- 카테고리에서 상세로 들어갈 때는 브라우저의 이전 스크롤 위치가 아니라 핵심 애니메이션 필드가 먼저 보여야 합니다.

## 추천 정보 구조

### 1. Einstieg / Szenarien

역할:

- 카테고리 선택 시작점
- 6개 구현 카테고리 중 하나를 바로 고르게 하는 모바일 보드

보여줄 요소:

- `Defensivfokus wählen.`
- 6개 카테고리 카드
- 카드 제목은 영어 야구 용어 가능
- 카드 보조 설명은 독일어
- 카드별 구현 시나리오 수

보여주지 않는 요소:

- 최근 학습 재진입
- 복잡한 메타 정보
- 긴 설명 블록

### 2. Ablauf

역할:

- 메인 학습 화면
- 전체 수비 움직임과 팀 콜 이해
- 상세 진입 시 가장 먼저 보여야 하는 기본 포커스 영역

보여줄 요소:

- 필드 SVG
- 필드 내부 경기 상태 HUD
- 9명 수비수
- 공 이동 애니메이션
- 재생 / 일시정지 토글
- 현재 phase와 진행 바
- 팀 콜 / Primary Out / Fallback 요약

보여주지 않는 요소:

- 속도 조절
- 별도 `Neu` 버튼
- `Quiz`
- `Coach`

### 3. Position

역할:

- 특정 포지션 상세 학습
- 필드를 본 직후 바로 역할을 확인하는 두 번째 핵심 화면

보여줄 요소:

- 선택 포지션 강조
- 시작 위치
- 첫 스텝
- 주요 책임
- 송구 우선순위
- 커버 / 백업
- 흔한 실수

모바일에서는 긴 하단 시트보다 `Ablauf`와 분리된 별도 모드 전환이 더 자연스럽습니다.

## 현재 MVP에서 제외하는 항목

- `Quiz`
- `Coach`

이유:

- 현재 단계에서는 `평가`보다 `장면 이해`와 `포지션 책임 이해`가 더 중요합니다.
- 작은 화면에서 탭 수를 늘리면 집중이 분산되고 화면 전환 비용이 커집니다.
- 코어 학습 흐름을 먼저 선명하게 만든 뒤, 확장 기능 여부를 다시 판단합니다.

## 추천 모바일 네비게이션

- 첫 진입 화면: 6개 카테고리 보드
- 두 번째 화면: 선택한 카테고리의 시나리오 카드 목록
- 상세 화면 모드 전환: `Ablauf`, `Position`
- 시나리오 전환은 카테고리 화면으로 복귀해 다시 선택합니다.
- 카테고리에서 상세로 들어갈 때는 핵심 애니메이션 필드가 바로 보이도록 포커스를 이동합니다.

## 첫 화면 와이어프레임

```text
------------------------------------------------
| Defensivfokus wählen.                        |
------------------------------------------------
| Infield          | Double Play              |
| kurze Erklärung  | kurze Erklärung          |
| 6 Übungen        | 1 Übung                  |
------------------------------------------------
| Bunt             | Play at the Plate        |
| kurze Erklärung  | kurze Erklärung          |
| 1 Übung          | 1 Übung                  |
------------------------------------------------
| Relay Home       | Pop-Up                   |
| kurze Erklärung  | kurze Erklärung          |
| 1 Übung          | 1 Übung                  |
------------------------------------------------
```

## 상세 화면 와이어프레임

```text
------------------------------------------------
| Zur Kategorie / Kategoriename                |
| Szenariotitel                                |
------------------------------------------------
|              Animated Field                  |
|         game-state HUD inside field          |
------------------------------------------------
| Aktive Phase / Progress      Play-Pause      |
------------------------------------------------
| P | C | 1B | 2B | SS | ...                   |
------------------------------------------------
| Ablauf | Position                            |
------------------------------------------------
| Team Call / Primary Out / Fallback          |
| oder                                         |
| Position: erster Schritt / Aufgabe / Wurf   |
------------------------------------------------
```

`Position` 모드에서는 같은 필드 프레임을 유지하되, 메인 정보 패널만 포지션 역할 중심으로 바뀝니다.

## 디자인 메모

- 버튼은 엄지로 누르기 쉽게 크게 만듭니다.
- 포지션 색상과 공의 이동 색상은 명확히 구분합니다.
- 애니메이션보다 이해도를 우선합니다.
- 작은 화면에서는 설명을 한 번에 길게 보여주지 않습니다.
- 필드를 보다가 계속 아래로 스크롤하게 만들지 않습니다.
- 독일어 라벨이 길어질 수 있으므로 모드 전환 버튼과 포지션 칩 폭을 여유 있게 잡습니다.
- 홈 카드는 아이콘보다 텍스트 정보가 더 중요하므로, 제목과 보조 설명만으로도 선택 이유가 읽혀야 합니다.
